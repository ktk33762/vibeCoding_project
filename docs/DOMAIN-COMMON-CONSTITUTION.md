# 공통 도메인 원칙

## 원칙

- 모든 도메인은 명확한 책임 경계를 가진다
- 모든 테이블은 RLS를 활성화한다
- 모든 PK는 `gen_random_uuid()`로 생성되는 UUID를 사용한다
- `created_at`은 모든 테이블에 포함하며 `DEFAULT now()`를 사용한다
- SELECT 권한은 도메인 특성에 따라 공개/제한을 결정한다. 결정 기준:
  - 공개 콘텐츠(게시글, 프로필, 팔로우) → SELECT 전체 공개
  - 개인 데이터(todos, 운동 목표/기록) → SELECT도 본인만 (단, 다른 사용자에게 보여줘야 하는 경우 공개로 변경)
