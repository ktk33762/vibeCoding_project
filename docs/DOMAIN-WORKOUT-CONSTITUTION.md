# 운동 도메인 헌법

## 목적

사용자가 운동 루틴을 템플릿으로 저장하고, 목표를 설정하며, 달성 현황을 추적할 수 있도록 지원한다.

## 핵심 개념

- **루틴 템플릿**: 반복 사용할 운동 목록의 집합. 이름과 운동 항목으로 구성된다.
- **운동 목표**: 주간 또는 월간 운동 횟수 목표. 사용자당 기간 유형별 하나만 유지한다.
- **운동 기록(Log)**: 실제 운동을 완료한 날짜와 루틴명의 기록. 사용자가 수동으로 등록한다.
- **운동 기록 상세 항목(Log Items)**: 해당 기록에서 수행한 개별 운동 내역 (종목, 세트, 횟수, 무게 등).
- **휴식 타이머**: 세트 간 휴식을 측정하는 클라이언트 사이드 기능. 서버 상태와 무관하다.

## 불변 원칙

- 루틴 템플릿은 사용자 본인만 생성·수정·삭제할 수 있다.
- 운동 기록 및 상세 항목은 사용자 본인만 생성·삭제할 수 있다.
- 운동 목표는 기간 유형(주간/월간)별로 하나만 존재하며, 새로 설정하면 기존 값을 덮어쓴다.
- 운동 목표와 기록은 SELECT가 공개되어 있어 프로필 페이지에서 다른 사용자의 운동 현황 조회가 가능하다.
- 앱에서 본인 데이터만 조회할 때는 반드시 `.eq('user_id', user.id)` 필터를 명시한다.
- 루틴 템플릿에서 할 일을 불러오면 기존 Todo 도메인의 데이터 구조를 그대로 사용한다.
- 타이머는 로컬 상태로만 동작하며 백엔드와 동기화하지 않는다.
