# 소셜 도메인 헌법

## 목적

사용자가 프로필을 갖고 서로를 팔로우하며, 팔로잉한 사람들의 게시글을 피드로 확인할 수 있도록 한다.

## 핵심 개념

- **프로필**: 사용자의 공개 정보. 닉네임, 소개글(bio), 팔로워/팔로잉 수, 게시글 목록, 운동 현황으로 구성.
- **팔로우**: 특정 사용자를 구독하는 단방향 관계. A가 B를 팔로우해도 B는 A를 팔로우하지 않을 수 있다.
- **팔로워/팔로잉 목록 모달**: 팔로워 수 또는 팔로잉 수를 클릭하면 해당 사용자 목록을 모달로 표시.
- **팔로잉 피드**: 게시판 내 "팔로잉" 탭으로 통합 — `/board`의 탭으로 존재하며 `/feed` 경로는 제거.

## 불변 원칙

- 프로필은 회원가입 시 자동 생성되며, 본인만 수정할 수 있다.
- 자기 자신을 팔로우할 수 없다.
- 팔로우 관계는 (팔로워, 팔로잉) 쌍으로 유일하다 — 중복 팔로우 불가.
- 팔로우/언팔로우는 로그인한 사용자만 할 수 있다.
- 프로필 조회와 팔로워/팔로잉 수 조회는 비로그인 사용자도 가능하다.
- 소셜 콘텐츠는 기존 게시판(posts) 데이터를 재사용한다. 별도 소셜 포스트 테이블을 두지 않는다.
- `/feed` 경로는 `/board`로 리다이렉트 — 팔로잉 피드가 게시판 탭으로 통합되었다.
