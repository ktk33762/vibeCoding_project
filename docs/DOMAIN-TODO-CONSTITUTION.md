# 할 일 도메인 헌법

## 할 일 개념

- 할 일(Todo)은 사용자가 관리하는 작업 단위이다.
- 각 할 일은 선택적으로 메모와 체크리스트 항목(TodoItem)을 포함할 수 있다.
- 할 일은 특정 회원에게 귀속되며 본인만 접근할 수 있다.

## 책임 범위

- 할 일 생성, 조회, 수정, 삭제 (CRUD)
- 체크리스트 항목 생성, 조회, 수정, 삭제
- 할 일 완료 상태 관리

## 원칙

- 할 일 데이터는 소유자(user_id)만 읽고 쓸 수 있다. (RLS 필수)
- 할 일이 삭제되면 소속 체크리스트 항목도 함께 삭제된다. (CASCADE)
- 체크리스트 항목의 순서는 sort_order 필드로 관리한다.
